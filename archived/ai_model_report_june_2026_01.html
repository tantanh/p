Bottom line

What changed, and what to do about it

  1. GLM-5.1 → GLM-5.2 everywhere.
    Verified, cheaper-per-quality, same price. No reason to stay on 5.1.
  2. Qwen 3.7 Max retired from the roster.
    GLM-5.2 now matches its context and beats its coding scores at a third of the output cost.
  3. Kimi K2.7-Code added as a watch-list Main (Table 1 only) for Code Simplifier.
    The token-efficiency story is compelling, but run your own TypeScript test in Kilo before trusting it on real refactors — the vendor benchmarks haven't been independently reproduced yet.
  4. Kimi K2.6 stays as the safe default everywhere else.
    Still the proven general-purpose anchor; K2.7-Code is explicitly Moonshot's coding-specialized successor, not a replacement for it.
One practical next step

Since GLM-5.2 is brand new and untested by Kilo's own TypeScript build suite, run one real build in your monorepo before routing production work to it — the same caution that caught DeepSeek V4 Pro's build failure earlier in this thread.
